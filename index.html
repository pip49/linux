2. 쉘(Shell)의 특징 및 환경 변수 제어

리눅스 기본 셸인 Bash(Bourne Again Shell)의 주요 기능과 단골 환경 변수 목록입니다.

  • Bash 쉘의 주요 기능: History 기능(기존 명령어 기억), Alias 기능(별명 지정), Tab 키 자동완성, 파이프 및 리다이렉션 지원.
  • 리다이렉션(Redirection) 기호 구별:
    • 명령어 > 파일 : 명령의 표준 출력을 파일로 보냄 (기존 내용 덮어쓰기).
    • 명령어 >> 파일 : 명령의 표준 출력을 파일에 추가함 (기존 내용 아래에 이어쓰기).
    • 명령어 < 파일 : 파일의 내용을 명령의 표준 입력으로 전달.
    • 명령어 2> 파일 : 명령 수행 중 발생한 표준 에러(Stderr)만 파일로 추출하여 저장.
  • 핵심 환경 변수 (대소문자 일치 필수):
    변수명의미 및 역할
    $PATH실행할 명령어를 검색할 디렉터리 경로 목록 (구분자는 콜론 :)
    $HOME현재 로그인한 사용자의 홈 디렉터리 절대 경로
    $USER / $UID현재 로그인한 사용자의 아이디 명칭 및 고유 UID 번호
    $SHELL사용자가 현재 사용 중인 기본 로그인 셸의 경로 (예: /bin/bash)
    $HISTSIZE명령어 히스토리 파일에 저장될 최대 명령어 개수 설정

3. 텍스트 검색 및 파일 편집 (심화 명령어)

  • grep 옵션 (텍스트 내 특정 패턴 검색):
    • -i : 검색어의 대소문자를 구분하지 않음.
    • -v : 매칭되지 않는(일치하지 않는) 라인만 반전하여 출력.
    • -n : 일치하는 텍스트가 위치한 행 번호(Line number)를 함께 표시.
    • -r : 하위 디렉터리 내부 파일 내용까지 재귀적으로 모두 검색.
  • find 옵션 (조건별 파일 검색):
    • find / -name "*.c" : 전체 경로에서 확장자가 .c인 파일 검색.
    • find / -user admin : 소유자가 admin인 파일 검색.
    • find / -perm 4755 : SetUID 권한(4000)이 부여된 파일 검색.

2과목. 운영 및 관리 (에디터, 시스템 관리, 파일시스템, 백업)

1. vi (Vim) 에디터의 3대 모드 및 단축키

vi 에디터는 실행 시 기본적으로 명령 모드로 시작합니다. 모드 전환 키와 단축키는 단골 문제입니다.

전환 명령 키이동 모드특징
i / a / o입력 모드 (Insert)i: 현재 커서 자리, a: 커서 다음 자리, o: 다음 행에 새로운 줄 삽입 후 입력
Esc명령 모드 (Command)커서 이동 및 텍스트 삭제/복사 수행 모드
: (콜론)ex 명령 모드 (Line)저장, 종료, 줄 번호 설정 및 문자열 치환 수행 모드

• ex 명령 모드 단축키 및 명령어

  • :w (저장), :q (종료), :wq! (강제 저장 후 종료), :q! (저장하지 않고 강제 종료)
  • :set nu (행 번호 표시), :set nonu (행 번호 숨기기)
  • 명령 모드 상태 동작: dd (현재 행 잘라내기/삭제), yy (현재 행 복사), p (붙여넣기), u (실행 취소/Undo)

2. 파일 시스템 빌드 및 디바이스 마운트 (fdisk -> mkfs -> mount)

새로운 디스크를 장착하고 시스템에 연결하는 일련의 행정 절차와 핵심 명령어 파트입니다.

# 1단계: 디스크 파티션 분할 작업 fdisk /dev/sdb (주요 서브명령: n=새 파티션, d=삭제, p=목록출력, w=저장종료) # 2단계: 파일시스템 포맷 (생성) mkfs -t ext4 /dev/sdb1 (또는 mke2fs -t ext4 명령어 동일 적용) # 3단계: 디렉터리와 디바이스 장치 연결 mount /dev/sdb1 /mnt/data
  • 리눅스 파일 시스템 유형별 특징:
    • ext2: 초기 리눅스 표준 파일시스템. 저널링 기능이 없어 비정상 종료 시 복구 시간 오래 걸림.
    • ext3: 저널링(Journaling) 기술 도입. 복구 속도 대폭 향상.
    • ext4: 현재 가장 널리 쓰임. 대용량 파일 지원 및 서브 디렉터리 개수 제한(64,000개) 확장.
    • xfs: SGI에서 개발한 고성능 64비트 저널링 파일시스템. RHEL 7버전 이후부터 기본 포맷으로 적용.

3. 사용자 계정 생성 정책 설정 파일 (고난도)

💡 계정 생성 시 적용되는 두 가지 기본값 파일:
useradd 명령어로 새로운 계정을 생성할 때 기본 홈 디렉터리 경로, 그룹 배정, 기본 셸, 만료일 등의 뼈대 설정은 다음 두 파일의 명세를 참조하여 자동 빌드됩니다.
  1. /etc/default/useradd: 기본 그룹 GID, 홈 디렉터리 상위 경로(/home), 계정 만료일(EXPIRE), 기본 셸 정의 파일.
  2. /etc/login.defs: 사용자의 UID/GID 시작 번호 및 끝 번호 범위 설정, 패스워드 최소/최대 사용 일수 설정, 메일 스풀 디렉터리 설정 파일.
  3. /etc/skel: 새로운 유저가 생성될 때 유저의 홈 디렉터리에 자동으로 복사되어 배치되는 환경 설정 뼈대 파일들이 모여 있는 디렉터리 (예: `.bashrc`, `.bash_profile` 등이 숨김 파일로 들어있음).

4. 예약 작업 자동화 (Cron & At)

  • cron: 주기적으로 영구 반복 실행되는 스케줄러. crontab -e 명령어로 편집하며, 필드 순서는 필수 암기 사항입니다.
    * * * * * [명령어 및 실행 스크립트 경로]
    분(0-59) 시(0-23) 일(1-31) 월(1-12) 요일(0-7, 0과 7은 일요일)
  • at: 주기적 반복이 아닌, 지정된 시간에 단 한 번만 실행되는 예약 스케줄러 (데몬명: atd, 설정 명령어: at [시간]).

2. 리눅스 패킷 필터링 방화벽 (iptables 심화)

  • 3대 기본 테이블 체인(Chain) 분류:
    • INPUT: 외부에서 시스템(서버 내부)으로 들어오는 패킷 검사.
    • OUTPUT: 시스템 자체에서 외부로 나가는 패킷 검사.
    • FORWARD: 시스템을 거쳐서 다른 목적지로 지나가는(라우팅되는) 패킷 검사.
  • 패킷 처리 타겟(Target) 처리 방식 종류:
    • ACCEPT: 패킷 통과 허용.
    • DROP: 패킷을 차단하고, 요청자에게 아무런 응답을 주지 않고 무시함 (스텔스 모드).
    • REJECT: 패킷을 차단하고, 요청자에게 명확한 거절 안내 패킷(ICMP 에러)을 반환함.
  • 명령어 사용 구문 예시:
    iptables -A INPUT -p tcp --dport 22 -j ACCEPT # TCP 22번(SSH) 포트로 들어오는 패킷 통과 허용
    iptables -A INPUT -p tcp --dport 80 -j DROP # TCP 80번(Web) 포트로 들어오는 패킷 차단 및 응답 안 함

3. 오픈소스 가상화 및 클라우드 기술 키워드

  • 가상화 하이퍼바이저 유형:
    • 전가상화(Full Virtualization): 하드웨어를 완전 가상화하여 게스트 OS 수정 불필요하나 느림 (예: KVM, VirtualBox).
    • 반가상화(Para Virtualization): 게스트 OS의 커널 일부를 수정하여 속도를 향상시킴 (예: Xen).
  • 컨테이너 기술: 하이퍼바이저 없이 커널 레벨의 가상화(Namespaces, cgroups)를 통해 가볍고 빠르게 격리된 공간을 만드는 기술 (대표 주자: Docker).
  • 클라우드 서비스 분류 모델 (필수 기본 상식):
    • IaaS (Infrastructure as a Service): 서버, 스토리지, 네트워크 등 하드웨어 자원만 인프라로 제공 (예: AWS EC2).
    • PaaS (Platform as a Service): 개발자가 코딩에만 집중할 수 있도록 개발 플랫폼 및 런타임 환경 제공 (예: Firebase, Heroku).
    • SaaS (Software as a Service): 완성된 소프트웨어를 웹을 통해 클라이언트에 제공 (예: 구글 드라이브).
