PART 01. 리눅스 일반 (역사, 설치 및 기본 명령어)

1. 리눅스의 역사 및 오픈소스 라이선스

  • 리눅스의 기원: 리누스 토발즈(Linus Torvalds)가 미닉스(Minix)를 기반으로 개발함. GNU 프로젝트의 철학과 결합됨.
  • GPL (General Public License): 자유 소프트웨어 재단(FSF)의 리차드 스톨만이 제창. 수정한 소스코드도 무조건 공개해야 하는 가장 엄격한 라이선스. (Linux 커널, Git 등)
  • LGPL (Lesser GPL): 독점 소프트웨어와 링크할 때는 소스코드 공개 의무 면제.
  • BSD / Apache 라이선스: 소스코드 수정 후 공개하지 않고 상용화 가능한 비교적 자유로운 라이선스.

2. FHS (Filesystem Hierarchy Standard) 표준 디렉터리 구조

경로주요 역할 및 특징 (시험 출제 100%)
/etc시스템 및 프로그램의 모든 환경 설정 파일이 위치함.
/var로그 파일(log), 프린터 스풀, 메일 등 시스템 운영 중 동적으로 크기가 변하는 파일 저장.
/bin부팅 및 단일 사용자 모드에서도 사용 가능한 일반 사용자용 기본 명령어 (ls, cp 등).
/sbin시스템 검사 및 복구 등 관리자(root) 전용 명령어가 위치함 (ifconfig, shutdown 등).
/usr윈도우의 Program Files와 유사. 설치된 응용 프로그램의 바이너리, 라이브러리, 문서 저장.
/proc실제 하드디스크 공간을 차지하지 않는 가상 파일 시스템. 커널 정보 및 프로세스 상태 정보 관리.
/dev마우스, 하드디스크, CD-ROM 등 시스템의 장치 파일들이 위치하는 곳.

3. 기본 디렉터리 조작 및 권한 명령어 옵션 상세

⚠️ 재귀적(Recursive) 옵션 대문자/소문자 구별 주의!
디렉터리와 그 하위 내용을 한 번에 처리할 때 cp, rm, chmod는 소문자 -r을 쓰고, chown, chgrp는 대문자 -R을 씁니다. 시험에서 매년 오답 유도로 출제됩니다.
  • ls 옵션:
    • -a : 숨김 파일(파일명이 .으로 시작)까지 포함하여 출력.
    • -l : 파일 권한, 소유자, 크기, 수정 시간 등을 상세히 출력.
    • -R : 하위 디렉터리 목록까지 재귀적으로 모두 출력.
    • -i : 파일의 고유 식별 번호인 인덱스 노드(i-node) 번호 출력.
  • cp 옵션: -r(디렉터리 복사), -p(소유주, 그룹, 권한, 시간 정보를 유지하며 복사), -a(-dpR과 동일, 아카이브 목적).
  • rm 옵션: -f(강제 삭제, 묻지 않음), -r(디렉터리 및 하위 내용 삭제).

4. 파일 소유권 및 권한 계산 (Permissions)

파일 상세 보기(ls -l)를 했을 때 나오는 10자리 문자 분해:

- rwx r-x r-- -> [파일타입][소유자(User)][그룹(Group)][기타사용자(Other)]
계산법: r=4, w=2, x=1 => rwx(4+2+1=7), r-x(4+0+1=5), r--(4+0+0=4) => 권한 숫자 '754'
  • 특수 권한 (시험 단골 고난도 문제):
    • SetUID (숫자 4000): 실행 시 소유자 권한으로 실행됨. (User 권한의 x자리에 s 표시)
    • SetGID (숫자 2000): 실행 시 그룹 권한으로 실행됨. (Group 권한의 x자리에 s 표시)
    • Sticky Bit (숫자 1000): 공유 디렉터리로 쓰이며 누구나 생성 가능하지만, 본인이 생성한 파일만 삭제 가능함. 예: /tmp 디렉터리 (Other 권한의 x자리에 t 표시)

PART 02. 리눅스 운영 및 관리 (사용자, 프로세스, 파일시스템, 패키지)

1. 사용자 계정 및 패스워드 관리 파일 구조 (필수 암기)

1) /etc/passwd 구조: 총 7개 필드로 구성, 콜론(:)으로 구분.

아이디 : 패스워드암호화(x) : UID : GID : 설명(Comment) : 홈디렉터리 : 기본 셸
예시) admin:x:1001:1001:System Administrator:/home/admin:/bin/bash

2) /etc/shadow 구조: 비밀번호 만료 정책 및 암호화 해시 관리.

아이디 : 암호화된패스워드 : 최종변경일 : 최소사용기간 : 최대사용기간 : 경고일수 : 비활성일수 : 만료일

2. 프로세스 관리 및 시그널(Signal)

  • ps 옵션 차이:
    • BSD 진영 계열 (하이픈 없음): ps aux (a: 모든 사용자, u: 사용자 중심 출력, x: 터미널이 없는 프로세스도 출력)
    • System V 진영 계열 (하이픈 필수): ps -ef (-e: 모든 프로세스, -f: 풀 포맷 출력)
  • 주요 kill 시그널 번호와 명칭:
    시그널 번호시그널 명칭특징
    1SIGHUP프로세스 재시작 (설정 파일을 다시 읽어옴)
    9SIGKILL가장 강력한 강제 종료 (프로세스가 거부 불가)
    15SIGTERM기본 종료 요청 (프로세스를 정상적으로 정리하고 종료 - kill 기본값)
  • 작업 제어: 백그라운드 전환 키(Ctrl + Z), 백그라운드 작업을 포그라운드로 가져오기(fg %작업번호).

3. 파일 시스템 및 디스크 관리

  • /etc/fstab의 6개 필드 구조: [장치명/UUID] [마운트포인트] [파일시스템타입] [옵션] [덤프여부(0 또는 1)] [파일시스템체크순서(0, 1, 2)]
  • 명령어 구별:
    • df: 시스템의 디스크 전체 남은 공간 확인 (-h 옵션은 인간이 읽기 편한 단위인 GB/MB로 출력).
    • du: 특정 디렉터리나 파일이 차지하는 용량 확인.

4. 패키지 관리 및 압축 아카이브

1) tar 명령어 옵션 (무조건 한 문제 나옴):

옵션역할옵션 조합 실무 예시
-c새로운 아카이브 파일 생성 (Create)ㆍ압축하기: tar -zcvf 백업.tar.gz 디렉터리
ㆍ압축 풀기: tar -zxvf 백업.tar.gz
-x아카이브 파일 추출 및 풀기 (eXtract)
-v압축/해제 과정을 화면에 상세히 출력 (Verbose)
-f대상 아카이브 파일명 지정 (File - 맨 뒤에 위치해야 함)
-zgzip 압축방식 연동 (.tar.gz 또는 .tgz)
-jbzip2 압축방식 연동 (.tar.bz2)
-Jxz 압축방식 연동 (.tar.xz)

PART 03. 리눅스 활용 (네트워크 설정, 오픈소스 서비스 및 인프라)

1. 네트워크 인터페이스 설정 및 설정 파일

  • 네트워크 인터페이스 활성화/비활성화: ifup eth0 / ifdown eth0
  • 기초 네트워크 인프라 확인 파일 정보:
    • /etc/sysconfig/network-scripts/ifcfg-[인터페이스명]: CentOS/RHEL 계열의 고정 IP, 서브넷 마스크, 게이트웨이 등을 직접 지정하는 설정 파일.
    • /etc/resolv.conf: 시스템이 참조할 DNS 서버의 IP 주소를 설정하는 파일 (예: nameserver 8.8.8.8).
    • /etc/hosts: DNS 서버를 거치지 않고 IP 주소와 도메인 이름을 로컬에서 수동으로 매핑해 두는 파일.

2. 네트워크 상태 점검 필수 명령어

  • netstat 옵션 조합 (핵심): netstat -anp 또는 netstat -lntp
    • -a : 모든 소켓 정보 출력
    • -n : 도메인 이름 대신 숫자(IP 주소/포트번호)로 직접 표시
    • -t : TCP 프로토콜 정보만 표시
    • -p : 해당 포트를 사용하는 프로세스 ID(PID)와 프로그램 명칭 표시
  • route 또는 ip route: 시스템의 라우팅 테이블(Gateway 정보) 확인 및 추가/삭제.
  • arp: 동일 네트워크 대역 내 하드웨어 주소(MAC 주소) 정보를 확인하는 명령어.

3. 주요 오픈소스 서버 서비스 및 네트워크 포트

🔥 서비스명 - 포트번호 - 핵심 설정 파일 경로는 통째로 연동하여 외우셔야 합니다!
서비스 종류프로토콜 및 기본 포트주요 데몬(Process) 및 환경 설정 파일 경로
Standalone / xinetd 구동 방식 분류 ㆍStandalone: 항상 메모리에 상주하며 직접 요청 처리 (Web, SSH 등)
ㆍxinetd: 관리 대행 데몬을 거쳐 실행되는 방식 (Telnet 등)
SSH (보안 원격 제어) TCP 22번 데몬: sshd
설정파일: /etc/ssh/sshd_config
Apache (웹 서버) TCP 80번 데몬: httpd 또는 apache2
설정파일: httpd.conf (기본 홈디렉터리 지시어: DocumentRoot)
Samba (이기종간 파일 공유) TCP 139, 445 / UDP 137, 138 리눅스와 윈도우 PC 간의 파일/프린터 공유 서비스
설정파일: /etc/samba/smb.conf
NFS (리눅스간 파일 공유) RPC 포트 바인딩 사용 리눅스 시스템끼리 특정 디렉터리를 원격 마운트해 공유
설정파일: /etc/exports
BIND (DNS 서버) UDP / TCP 53번 도메인을 IP로 변환.
설정파일: /etc/named.conf, 존(Zone) 파일 설정 필요.
